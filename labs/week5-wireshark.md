# Week 5 – Wireshark Packet Analysis

## Objective

To analyse network traffic and identify security vulnerabilities in HTTP communication.

## What I Did

* Captured network packets using Wireshark
* Applied filters to analyse HTTP POST requests
* Identified transmitted credentials in plaintext

## Key Observation

* Username and password were visible in HTTP packets
* Data was not encrypted

## Learning Outcome

This experiment demonstrated that HTTP is insecure and sensitive information can be intercepted easily using packet sniffing tools.

## Tools Used

* Wireshark
